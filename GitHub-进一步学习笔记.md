# GitHub 进一步学习

### **GitHub 上的协作**

- **Fork（派生）仓库**：可以将其他人的仓库复制到自己的 GitHub 账户下，<u>进行修改并提交 Pull Request（**PR**）请求，邀请原仓库的维护者进行合并</u>。
- **Pull Request（PR）**：PR 是用于请求将你在分支上的更改合并到主分支（例如 `master` 或 `main`）的机制。<u>其他开发者可以对 PR 进行审查、评论并合并。</u>
- **Issue（问题追踪）**：你可以<u>创建 Issues 来跟踪 bug 或新的功能请求</u>。==Issues 可以用来讨论、分配任务和解决问题==。
- **Wiki 和文档**：GitHub 支持在仓库中创建 Wiki [^1]和文档，方便项目的协作和知识共享。

### **GitHub 项目管理功能**

GitHub <u>提供了项目管理的功能，帮助团队更好地跟踪进度和安排工作</u>：

- **Projects**：可以创建看板（Kanban board），使用任务卡片来管理和分配工作。
- **Actions**：GitHub Actions 是一个 CI/CD 工具，可以自动化构建、测试、部署工作流。
- **Discussions**：用于项目成员之间的讨论，类似于论坛的形式，适合用于交流和提问。

### **GitHub Pages**

GitHub Pages 是 GitHub 提供的免费静态网页托管服务，可以将项目以网页的形式展示出来。只需在项目的 `gh-pages` 分支上上传 HTML、CSS 和 JavaScript 文件，GitHub 会自动为我们生成网页。

### **开源项目**

GitHub 是开源软件的中心，可以贡献代码或提交问题报告。常见的贡献流程如下：

- **Fork 仓库**：将项目复制到自己的 GitHub 账户。
- **Clone 本地**：将 Fork 后的仓库克隆到本地进行开发。
- **修改并提交**：修改代码后提交变更，并推送到你的 GitHub 仓库。
- **创建 Pull Request**：提交 PR 请求，等待维护者审核和合并。
