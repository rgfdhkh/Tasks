# GitHub 入门笔记：

## 什么是 GitHub

**GitHub 是一个面向开源及私有软件项目的托管平台，主要功能包括：**

1. **代码托管：**
   可以安全地存储你的代码。
2. **版本控制：**
   使用 Git 进行版本管理，方便跟踪代码的变化。
3. **协作开发：**
   多人可以共同参与一个项目。

## 注册与登录

1. 访问官网（github.com），点击“Sign up”进行注册，填写用户名、邮箱、密码等信息。
2. 注册成功后，使用用户名和密码登录。

## 创建仓库（Repository）

1. 登录后，点击右上角的“+”，选择“New repository”。
2. 填写仓库名称、描述等信息，选择仓库的可见性（公开或私有）。
3. 可以选择添加.gitignore 文件（用于忽略不需要提交的文件）和许可证。
4. 点击“Create repository”完成创建。

## 克隆仓库到本地

1. 在仓库页面找到“Clone or download”按钮，点击后复制仓库的 URL。
2. 打开命令行工具，使用“git clone [仓库 URL]”命令将仓库克隆到本地。

## 提交代码

1. 在本地仓库中进行代码修改。
2. 使用“git add [文件名]”将修改的文件添加到暂存区。
3. 使用“git commit -m "提交说明"”提交修改，提交说明应简洁明了地描述本次修改的内容。
4. 使用“git push”将本地代码推送到远程仓库。

## 分支管理

1. 创建分支：
   使用“git branch [分支名]”创建新分支。
2. 切换分支：
   使用“git checkout [分支名]”切换到指定分支。
3. 合并分支：
   在主分支上使用“git merge [分支名]”将其他分支的代码合并到主分支。

## 协作开发

1. 邀请其他开发者：
   在仓库的“Settings”中找到“Collaborators”，输入对方的 GitHub 用户名进行邀请。
2. 处理拉取请求（Pull Request）：
   其他开发者提交代码变更时会创建拉取请求，你可以审查代码并决定是否合并。

## 问题跟踪（Issues）

1. 创建问题：
   用于报告 bug、提出功能需求等。
2. 分配问题：
   将问题分配给特定的开发者。
3. 讨论问题：
   开发者可以在问题页面进行讨论。

## 协作开发

1.  邀请团队成员加入项目。
2.  使用@mentions 通知团队成员。
3.  利用 GitHub Actions 自动化工作流程。

## 使用 GitHub Pages

1. 可以将仓库设置为 GitHub Pages，用于托管静态网站。
2. 在仓库的“Settings”中找到“Pages”，选择要部署的分支和目录。

总之，GitHub 是一个强大的代码托管和协作平台，通过掌握这些基本操作，你可以更好地利用它进行软件开发和团队协作。

## 分支管理

1.  创建分支：`git branch [branch-name]`创建新分支。
2.  切换分支：`git checkout [branch-name]`切换到指定分支。
3.  合并分支：`git merge [branch-name]`将指定分支合并到当前分支。

## Pull Request(拉取请求)

1.  在 GitHub 仓库页面，点击Pull requests。
2.  点击New pull request，选择要合并的分支。
3.  检查更改，添加评论，然后点击Merge pull request合并。

## 复制(探索)与贡献

1.  Fork（复制）：复制别人的仓库到自己的 GitHub 账号下。
2.  Contribute（贡献）：对复制的仓库进行更改，然后发起 Pull Request 贡献代码。
